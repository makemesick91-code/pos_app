# UIX-2 Responsive Validation

Chrome headless captures were rendered at 360×800, 390×844, 412×915, 768×1024, 1024×768, 1280×800, 1440×900, and 1920×1080. Evidence is in `docs/uiux/evidence/uix-2/`.

Desktop uses a balanced split hero and three-column content. Tablet collapses the hero, adapts navigation, and uses two-column cards. Mobile uses one-column hierarchy, full-width CTAs, a 46 px menu control, scaled device preview, vertical steps, and single-column pricing. Visual inspection found no hero clipping, horizontal overflow, overlapping content, missing icons, or hidden CTA.
