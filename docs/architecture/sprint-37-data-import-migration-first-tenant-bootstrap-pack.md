# Sprint 37 Data Import, Migration & First Tenant Bootstrap Pack

Sprint 37 adds a governed platform-admin import foundation for CSV tenant bootstrap data. It supports dry-run validation and explicit execution for category, product, supplier, customer, initial stock, price, payment method, default settings, and bootstrap-pack flows.

Commercial chain:
Plan → Invoice → Payment Intent → Gateway Settlement → Collection → Entitlement Runtime Access → Tenant Onboarding → Android Runtime → Support Operations → Observability → Data Import/Bootstrap.

Non-goals: no Android import UI, no tenant-admin mutation route, no XLSX parser dependency, no payment gateway credential import, no direct stock balance mutation, no direct invoice/payment/entitlement mutation.

Architecture:
- `TenantDataImportService` is the mutation orchestrator (`IMP-R001`).
- `ImportFileParserService` parses CSV and rejects XLSX/XLSM/XLS with a governed reason (`IMP-R008`, `IMP-R009`).
- `ImportValidationService` validates headers and rows before mutation (`IMP-R010`, `IMP-R011`).
- `ImportIdempotencyService` hashes run keys, files, and normalized rows (`IMP-R005`, `IMP-R006`, `IMP-R007`).
- Domain handlers import categories, products, suppliers, customers, initial stock, prices, payment methods, and settings tenant-scoped (`IMP-R014`..`IMP-R021`).
- `InitialStockImportService` uses `InventoryMovementService` opening movements and import references (`IMP-R018`, `IMP-R019`).
- `ImportRollbackService` deletes only import-created non-ledger records and reverses stock through inventory movement (`IMP-R013`).
- `ImportAuditService` writes redacted admin audit records (`IMP-R023`, `IMP-R026`, `IMP-R032`).
- `ImportSupportBridgeService` and `ImportObservabilityBridgeService` expose safe summaries/signals (`IMP-R024`, `IMP-R025`).

File policy:
- CSV is canonical and generated by `import:template`.
- XLSX/XLSM/XLS is deferred because no safe lightweight parser is installed; formulas/macros are never evaluated.
- Raw file contents, raw paths, PII, and secrets are not printed by commands, resources, audit, smoke, or tests.

Rollback:
Run `import:rollback --run=<id>` for dry-run. Add `--execute --reason=<code>` to mutate. Rollback affects import-created subjects only; stock rollback writes a reversal movement.
