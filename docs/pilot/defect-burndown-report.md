# Pilot Defect Burn-down Report (Sprint 17)

Generated by `php artisan pilot:burndown-summary [--json] [--strict]` and
`DefectBurnDownService::summary()`. Read-only; never prints secrets.

## Daily burn-down table

| Date       | Total | Open | Fixed | Retest | Verified | Closed | Accepted risk | SLA breached | Decision |
|------------|-------|------|-------|--------|----------|--------|---------------|--------------|----------|
| YYYY-MM-DD | 0     | 0    | 0     | 0      | 0        | 0      | 0             | 0            | GO       |

Fill one row per pilot day from the command output.

## Count by severity

Counts of all defects grouped by `severity` (BLOCKER / CRITICAL / MAJOR / MINOR /
TRIVIAL).

## Count by status

Counts grouped by `status` (OPEN / IN_PROGRESS / FIXED / RETEST / VERIFIED / CLOSED /
ACCEPTED_RISK).

## Count by area

Counts grouped by `area` (AUTH, SYNC, CASHIER, PAYMENT_QRIS, RECEIPT_PRINTER,
OFFLINE_SYNC, INVENTORY, REPORTING, CLOSING, SUBSCRIPTION_DEVICE, ADMIN_ONBOARDING,
ANDROID_APP, BACKEND_API, OTHER).

## SLA breach count

`counts.sla_breached_open` — open defects past their severity SLA. See
[sla-breach-detection.md](sla-breach-detection.md).

## Accepted risk count

`counts.accepted_risk`, plus `accepted_risk_blocking_valid` and
`accepted_risk_blocking_expired`. See [accepted-risk-governance.md](accepted-risk-governance.md).

## Retest / verified / closed count

`counts.retest`, `counts.verified`, `counts.closed` track the fix verification
pipeline. See [fix-verification-retest-workflow.md](fix-verification-retest-workflow.md).

## GO / WATCH / NO-GO interpretation

- **NO-GO** — an open BLOCKER/CRITICAL without valid accepted risk, or an
  accepted-risk BLOCKER/CRITICAL whose acceptance has expired.
- **WATCH** — an open MAJOR defect, or a validly accepted blocking/major defect.
- **GO** — only MINOR/TRIVIAL remain open, or nothing is open.

`--strict` makes the command exit non-zero on WATCH (and always on NO-GO), so CI can
gate on it.
