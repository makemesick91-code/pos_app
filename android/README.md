# Aish POS Lite — Android App

Native **Android (Kotlin)** POS app for the Aish POS Lite SaaS.

> Source of truth: [`../docs/foundation/POS_ANDROID_SAAS_FOUNDATION.md`](../docs/foundation/POS_ANDROID_SAAS_FOUNDATION.md)

## Project Facts

| Item        | Value                     |
| ----------- | ------------------------- |
| Package     | `com.aishtech.poslite`    |
| App name    | Aish POS Lite             |
| Language    | Kotlin (native, XML/Views)|
| minSdk      | 26 (Android 8)            |
| targetSdk   | 35                        |
| compileSdk  | 35                        |

## Sprint 0 Scope

Sprint 0 only provides a **valid, launchable skeleton**:

- correct package name and SDK levels per the foundation
- a placeholder `MainActivity` that opens

No business POS features (cashier, product, sales, QRIS, offline sync, printer,
subscription) exist yet. Those are introduced in later sprints per the foundation
roadmap.

## Local Development Setup

Requirements:

- Android Studio (Ladybug or newer)
- Android SDK Platform 35, Build Tools
- JDK 17

Open the `android/` folder in Android Studio, let it sync Gradle, then run the
`app` configuration on an emulator or device (API 26+).

## Build

```bash
cd android
./gradlew :app:assembleDebug
```

> The Gradle wrapper (`gradlew`) is generated by Android Studio / a local Gradle
> install on first sync. It is intentionally not committed in Sprint 0 because no
> Android SDK/Gradle toolchain is provisioned in the current setup environment.
> See `docs/sprints/sprint-0-project-setup.md` for the documented tooling
> limitation — the build was **not** claimed as passing.
